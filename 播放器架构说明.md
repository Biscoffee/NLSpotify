## 🔄 核心协作模式：观察者模式（通知机制）

### 1. **通知系统**

NLPlayerManager 通过 `NSNotificationCenter` 向所有观察者广播状态变化：

```objective-c
// 三个核心通知
NLPlayerSongDidChangeNotification          // 歌曲变化
NLPlayerPlaybackStateDidChangeNotification // 播放状态变化
NLPlayerProgressDidChangeNotification      // 进度变化
```

### 2. **双向通信**

- **Controller → Manager**: 直接调用方法（命令模式）
- **Manager → Controller**: 通过通知（观察者模式）

## 🎵 功能协作详解

### 一、播放/暂停功能

#### 流程：

```
用户点击播放/暂停按钮
    ↓
Controller 调用 Manager.togglePlayPause()
    ↓
Manager 内部处理：
  - 如果正在播放 → pause()
  - 如果已暂停 → play()
    ↓
Manager 更新 playbackState
    ↓
Manager 发送 NLPlayerPlaybackStateDidChangeNotification
    ↓
两个 Controller 同时收到通知
    ↓
Controller 更新 UI（播放按钮图标）
```

#### 代码实现：

**大播放器 (NLMusicPlayerViewController)**:
```objective-c
- (void)musicPlayerViewDidTapPlayPause:(NLMusicPlayerView *)view {
    [NLPlayerManager.sharedManager togglePlayPause];
    // ↑ 直接调用 Manager
}

- (void)bindPlayer {
    // 监听播放状态变化
    [center addObserver:self selector:@selector(refreshPlayState)
                   name:NLPlayerPlaybackStateDidChangeNotification object:nil];
}

- (void)refreshPlayState {
    BOOL playing = NLPlayerManager.sharedManager.playbackState == NLPlaybackStatePlaying;
    [self.playerView updatePlayState:playing]; // 更新 UI
}
```

**小播放器 (NLMusicPlayerAccessoryView)**:
```objective-c
- (void)onPlayPauseTapped {
    [[NLPlayerManager sharedManager] togglePlayPause];
    // ↑ 同样的调用
}

- (void)onPlaybackStateChanged {
    [self refreshPlayButton]; // 更新 UI
}
```

**Manager (NLPlayerManager)**:
```objective-c
- (void)togglePlayPause {
    if (self.playbackState == NLPlaybackStatePlaying) {
        [self pause];
    } else {
        [self play];
    }
}

- (void)play {
    [self.player play];
    [self setPlaybackState:NLPlaybackStatePlaying]; // 发送通知
}

- (void)pause {
    [self.player pause];
    [self setPlaybackState:NLPlaybackStatePaused]; // 发送通知
}
```

### 二、切歌功能（上一首/下一首）

#### 流程：

```
用户点击上一首/下一首按钮
    ↓
Controller 调用 Manager.playPrevious() 或 playNext()
    ↓
Manager 计算新索引（支持循环）
    ↓
检查新歌曲是否有 playURL
    ├─ 有 → 直接播放
    └─ 无 → 异步获取 playURL → 播放
    ↓
Manager 更新 currentSong 和 currentIndex
    ↓
Manager 发送 NLPlayerSongDidChangeNotification
    ↓
两个 Controller 同时收到通知
    ↓
Controller 更新 UI（封面、标题、艺术家）
```

#### 代码实现：

**大播放器**:
```objective_c
- (void)musicPlayerViewDidTapPrevious:(NLMusicPlayerView *)view {
    [NLPlayerManager.sharedManager playPrevious];
}

- (void)musicPlayerViewDidTapNext:(NLMusicPlayerView *)view {
    [NLPlayerManager.sharedManager playNext];
}
```

**小播放器**:
```objective_c
- (void)onNextTapped {
    [[NLPlayerManager sharedManager] playNext];
}
```

**Manager**:
```objective_c
- (void)playNext {
    NSInteger nextIndex = self.currentIndex + 1;
    if (nextIndex >= self.playlist.count) {
        nextIndex = 0; // 循环播放
    }
    
    NLSong *nextSong = self.playlist[nextIndex];
    if (nextSong.playURL) {
        [self playSongAtIndex:nextIndex]; // 直接播放
    } else {
        // 异步获取 playURL
        [[NLSongService sharedService] fetchPlayableURLWithSongId:nextSong.songId
                                                         success:^(NSURL *playURL) {
            nextSong.playURL = playURL;
            [self playSongAtIndex:nextIndex];
        }];
    }
}

- (void)playSongAtIndex:(NSInteger)index {
    self.currentIndex = index;
    self.currentSong = self.playlist[index];
    [self postSongChangedNotification]; // 发送通知
    [self playWithURL:song.playURL];
}
```

### 三、进度调整功能

#### 流程：

```
用户拖拽进度条
    ↓
View 触发 didChangeProgress: 回调
    ↓
Controller 调用 Manager.seekToProgress()
    ↓
Manager 计算目标时间并跳转
    ↓
AVPlayer 跳转到新位置
    ↓
Manager 的 timeObserver 继续发送进度更新
    ↓
Controller 收到 NLPlayerProgressDidChangeNotification
    ↓
Controller 更新进度条（如果不在拖拽状态）
```

#### 代码实现：

**大播放器**:
```objective_c
// View 回调
- (void)musicPlayerView:(NLMusicPlayerView *)view didChangeProgress:(float)value {
    [[NLPlayerManager sharedManager] seekToProgress:value];
    // ↑ 直接调用 Manager
}

// 监听进度变化
- (void)bindPlayer {
    [center addObserver:self selector:@selector(playerProgressDidChange:)
                   name:NLPlayerProgressDidChangeNotification object:nil];
}

- (void)playerProgressDidChange:(NSNotification *)notification {
    if (self.playerView.isTrackingProgress) return; // 拖拽时不更新
    
    NSNumber *progressNum = notification.object;
    [self.playerView updateProgress:progressNum.floatValue];
    
    // 同时更新时间
    NSTimeInterval currentTime = NLPlayerManager.sharedManager.currentTime;
    NSTimeInterval totalTime = NLPlayerManager.sharedManager.totalTime;
    [self.playerView updateCurrentTime:currentTime totalTime:totalTime];
}
```

**Manager**:
```objective_c
- (void)seekToProgress:(float)progress {
    CMTime duration = self.player.currentItem.duration;
    CMTime targetTime = CMTimeMultiplyByFloat64(duration, progress);
    [self.player seekToTime:targetTime]; // AVPlayer 跳转
}

// 定时器发送进度更新（每0.5秒）
- (void)addPeriodicTimeObserver {
    self.timeObserver = [self.player addPeriodicTimeObserverForInterval:CMTimeMakeWithSeconds(0.5, NSEC_PER_SEC)
                                                                  queue:dispatch_get_main_queue()
                                                             usingBlock:^(CMTime time) {
        float progress = [self currentProgress];
        [[NSNotificationCenter defaultCenter] 
         postNotificationName:NLPlayerProgressDidChangeNotification 
         object:@(progress)];
    }];
}
```

**关键点**：
- 拖拽时：`isTrackingProgress = YES`，不更新进度条（避免冲突）
- 松开时：`isTrackingProgress = NO`，恢复自动更新

### 四、音量调整功能

#### 流程：

```
用户拖拽音量条
    ↓
View 触发 didChangeVolume: 回调
    ↓
Controller 直接设置 Manager.volume 属性
    ↓
Manager 的 setVolume: 方法被调用
    ↓
Manager 更新 AVPlayer.volume
    ↓
音量立即生效（无需通知）
```

#### 代码实现：

**大播放器**:
```objective_c
- (void)musicPlayerView:(NLMusicPlayerView *)view didChangeVolume:(float)value {
    [NLPlayerManager sharedManager].volume = value;
    // ↑ 直接设置属性
}
```

**Manager**:
```objective_c
@property (nonatomic, assign) float volume; // 可读写属性

- (void)setVolume:(float)volume {
    _volume = volume;
    self.player.volume = volume; // 立即生效
}
```

**关键点**：
- 音量是**属性直接设置**，不需要通知机制
- 因为音量变化是**即时反馈**，不需要同步到其他 UI
- 小播放器没有音量控制，所以不需要监听

### 播放/暂停数据流：

## 🔑 关键设计模式

### 1. **单例模式**
- `NLPlayerManager` 使用单例，确保全局只有一个播放器实例
- 所有 Controller 共享同一个播放状态

### 2. **观察者模式**
- Manager 通过通知广播状态变化
- Controller 订阅通知，自动更新 UI
- **解耦**：Controller 不需要直接持有 Manager 的引用

### 3. **命令模式**
- Controller 通过方法调用控制 Manager
- Manager 执行具体操作（播放、暂停、切歌等）

---

## 💡 协作优势

### 1. **解耦**
- Controller 之间不直接通信
- 都通过 Manager 间接交互
- 易于扩展和维护

### 2. **同步**
- 两个播放器 UI 自动同步
- 状态变化实时反映
- 无需手动同步代码

### 3. **统一管理**
- 所有播放逻辑集中在 Manager
- 单一数据源（Single Source of Truth）
- 避免状态不一致

---

## 📝 总结

### 核心协作关系：

1. **Manager 是中心**：所有播放逻辑和状态管理
2. **Controller 是观察者**：监听状态变化，更新 UI
3. **通知是桥梁**：Manager → Controller 的通信方式
4. **方法调用是命令**：Controller → Manager 的控制方式

### 数据流方向：

- **控制流**：Controller → Manager（方法调用）
- **状态流**：Manager → Controller（通知机制）
- **音量**：直接属性设置（即时生效）
- **进度**：定时器 + 通知（持续更新）

这样的设计确保了：
- ✅ 代码解耦
- ✅ 状态同步
- ✅ 易于维护
- ✅ 可扩展性强
