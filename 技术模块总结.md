# NLSpotify 技术模块总结（文件与方法级）

本文档对以下模块进行详尽总结，具体到每个文件的每个方法：  
双 Token、下载音乐、WCDB、缓存机制、预加载、我的歌单、首页 TableView 嵌套 CollectionView、高度自适应、评论区富文本高度适应。

---

## 一、双 Token（登录态与刷新）

### 1. NetWorkManager.m

| 方法 | 作用 |
|------|------|
| `+ requestHeadersWithCookie` | 从 `NLAuthManager` 取当前 cookie，组装为请求头 `@{ @"Cookie": cookie }`，无 cookie 时返回空字典。 |
| `+ sharedManager` | 单例，初始化时调用 `setupSessionManager`。 |
| `- init` | 初始化 `pendingRequests`、串行队列 `refreshLockQueue`。 |
| `- setupSessionManager` | 创建 AFHTTPSessionManager，设置 requestSerializer、responseSerializer、timeout 20s。 |
| `- forceLogoutAndFailWithError:failure:` | 调用 `NLAuthManager logout`，发 `NLForceLogoutNotification`，执行 failure。 |
| `- forceLogoutAndKickToLogin` | 登出并发强制登出通知。 |
| `- GET:parameters:success:failure:` | 对外 GET，内部调带 retryCount 的 GET（retryCount=0）。 |
| `- POST:parameters:success:failure:` | 对外 POST，内部调带 retryCount 的 POST（retryCount=0）。 |
| `- GET:parameters:retryCount:success:failure:` | 内部 GET：拼 URL、取 `requestHeadersWithCookie`，成功时若 bizCode==301 走 `handleAuthRequiredForGET`，失败时若 statusCode==401 也走该逻辑，否则回调 success/failure。 |
| `- POST:parameters:retryCount:success:failure:` | 内部 POST：同上，301/401 时走 `handleAuthRequiredForPOST`。 |
| `- handleAuthRequiredForGET:params:retryCount:success:failure:` | 封装为 GET，转调 `handleAuthRequiredWithPath:... isGET:YES`。 |
| `- handleAuthRequiredForPOST:params:retryCount:success:failure:` | 封装为 POST，转调 `handleAuthRequiredWithPath:... isGET:NO`。 |
| `- handleAuthRequiredWithPath:params:isGET:retryCount:success:failure:` | 双 Token 核心：若 retryCount≥kMaxRetryCount 则强制登出并 failure；否则构造 `NLPendingAuthRequest`，在 `refreshLockQueue` 上若正在刷新则把请求入队，否则设 `isRefreshing=YES`，主线程调 `NLAuthService refreshLoginWithSuccess:`，成功则清空 pending 并逐个 `retryRequest:withRetryCount:1`，失败则对当前及 pending 全部 failure 并 `forceLogoutAndKickToLogin`。 |
| `- retryRequest:withRetryCount:` | 按 req.isGET 调用 GET 或 POST，传入相同 path/params 与 retryCount。 |

### 2. NLAuthManager.h / NLAuthManager.m

| 方法 | 作用 |
|------|------|
| `+ currentCookie` | 从 UserDefaults 读 `kCookieKey`，返回当前 cookie 字符串。 |
| `+ setCookie:` | 有值则写入 UserDefaults，无值则移除；登录/游客登录/refresh 成功后调用。 |
| `+ setLoginStateWithAccount:` | 设置已登录状态，可选保存 account。 |
| `+ isLoggedIn` | 根据 UserDefaults 的 `kIsLoggedInKey` 返回是否已登录。 |
| `+ logout` | 清 cookie、清登录状态、清 userAccount。 |

常量：`NLForceLogoutNotification`；Key：`kCookieKey`、`kIsLoggedInKey`、`kUserAccountKey`。

### 3. NLAuthService.h / NLAuthService.m

| 方法 | 作用 |
|------|------|
| `+ refreshLoginWithSuccess:failure:` | 请求 `/login/refresh`（GET），成功且 code==200 时用返回的 `cookie` 调 `NLAuthManager setCookie:`，再调 success；否则 failure。 |

双 Token 流程简述：业务请求带 cookie → 若 301/401 则加锁、调 refresh → refresh 成功则写回 cookie 并重试原请求及队列中请求；失败则登出并踢回登录。

## 二、下载音乐

### 1. NLDownloadManager.h / NLDownloadManager.m

| 方法 | 作用 |
|------|------|
| `+ sharedManager` | 单例。 |
| `- init` | 创建 NSURLSession（defaultConfig，delegate=self）、taskContextMap、NSLock。 |
| `- addDownloadForSong:` | 若 song 无 songId 或已在 `NLDownloadRepository` 中则 return；若有 playURL 直接 `startDownloadWithSong:`；否则先 `NLSongService fetchPlayableURLWithSongId`，成功后再 `startDownloadWithSong:`。 |
| `- startDownloadWithSong:` | 调 `NLDownloadRepository addDownloadItemWithSong:status:@"downloading"`，发 `NLDownloadManagerDidUpdateNotification`，用 song.playURL 建 Range 请求，创建 NSURLSessionDataTask，在 taskContextMap 中存 url/songId/currentOffset/totalLength/sessionRanges，resume。 |
| `- isDownloadingSongId:` | 在 taskContextMap 中查找是否有 context 的 songId 等于该 songId。 |
| `- cancelAllDownloads` | session getTasksWithCompletionHandler，对全部 dataTasks cancel，清空 taskContextMap，主线程发 DidUpdate 通知。 |
| `- URLSession:dataTask:didReceiveResponse:completionHandler:` | 从 Content-Length 或 Content-Range 取 totalLength，写入对应 context，completionHandler(NSURLSessionResponseAllow)。 |
| `- URLSession:dataTask:didReceiveData:` | 从 context 取 url、currentOffset、totalLength，调 `NLCacheManager cacheData:forURL:atOffset:totalLength:`，把本次 range 加入 sessionRanges，更新 currentOffset。 |
| `- URLSession:task:didCompleteWithError:` | 从 map 移除该 task；若有 error 且非 NSURLErrorCancelled 只发通知；否则调 `mergeAndSaveSessionRanges`，若 `cachedRangesCoverFullLengthForURL` 则 `finishCacheForURL`，`NLDownloadRepository updateStatus:@"completed"`，构造 NLSong 并 `NLSongRepository addDownloadedSong:`，发 DidUpdate。 |

### 2. NLDownloadRepository.h / NLDownloadRepository.mm

| 方法 | 作用 |
|------|------|
| `+ safeCreateTable` | 若表不存在则 `createTable:kDownloadQueueTableName withClass:NLDownloadItem.class`。 |
| `+ allDownloadItems` | safeCreateTable 后从表取全部 NLDownloadItem，按 addedTime 新→旧排序返回。 |
| `+ downloadItemForSongId:` | 按 songId 查单条 NLDownloadItem。 |
| `+ addDownloadItemWithSong:status:` | 用 song 构造 NLDownloadItem（songId/playURLString/title/artist/coverURLString/addedTime/status），insertOrReplace 进表。 |
| `+ updateStatus:forSongId:` | 查出 item 后改 status 再 insertOrReplace。 |
| `+ removeDownloadItemWithSongId:` | deleteFromTable where songId == songId。 |
| `+ clearAllDownloadItems` | deleteFromTable 整表。 |

表名：`DownloadQueueTable`。

### 3. NLDownloadItem.h / NLDownloadItem.mm（ORM）

| 方法/宏 | 作用 |
|---------|------|
| `WCDB_IMPLEMENTATION` / `WCDB_SYNTHESIZE` | 绑定 songId、playURLString、title、artist、coverURLString、addedTime、status。 |
| `WCDB_PRIMARY(songId)` | 主键 songId。 |
| `- playURL` | playURLString 转 NSURL。 |

### 4. 使用下载的页面（简述）

- **NLMusicLibraryListViewController**：`NLMusicLibraryListModeDownloadedSongs` 下 loadData 调 `NLDownloadRepository allDownloadItems`，cell 用 `configWithDownloadItem:downloadProgress:`；滑删时先弹窗再删 `removeDownloadItemWithSongId` + `removeDownloadedSong`。
- **NLMusicPlayerViewController**：三点菜单「下载」调 `addDownloadForSong:`。
- **NLSongListViewController**：批量下载确认后对歌单每首 `addDownloadForSong:`。

## 四、缓存机制

### 1. NLCacheManager.h / NLCacheManager.mm

目录：`Caches/NLAudioCache`；表：`AudioCacheInfoTable`（NLAudioCacheInfo）。  
文件：未完成 `.tmp`，完成 `.mp3`（由 finishCacheForURL 重命名）。

| 方法 | 作用 |
|------|------|
| `+ sharedManager` | 单例。 |
| `- init` | 创建 NLAudioCache 目录、createTable AudioCacheInfoTable。 |
| `- md5StringFromURL:` | URL 转 MD5 字符串作文件名。 |
| `- tempFilePathForURL:` | 返回 `cacheDirectory/%@.tmp`。 |
| `- cacheFilePathForURL:` | 返回 `cacheDirectory/%@.mp3`。 |
| `- isFullyCachedForURL:` | 判断 .mp3 是否存在。 |
| `- totalLengthForURL:` | 从 DB 按 urlMD5 查 NLAudioCacheInfo.totalLength。 |
| `- cacheProgressForURL:` | 已整首则 1；否则用 cachedRanges 覆盖长度/ totalLength，上限 1。 |
| `- lastAccessTimeForFullyCachedURL:` | 整首缓存时返回 lastAccessTime。 |
| `- cachedRangesForURL:` | 从 DB 取 cachedRangesString 解析为 NSArray<NSValue(NSRange)>。 |
| `- queueForMD5:` | 每 URL(MD5) 对应一条串行 dispatch_queue，用于同文件并发安全。 |
| `- cacheData:forURL:atOffset:totalLength:` | 按需建 NLAudioCacheInfo 档案（totalLength、lastAccessTime），在 queueForMD5 队列内用 pwrite 写入 .tmp。 |
| `- mergeAndSaveSessionRanges:forURL:` | 在 queueForMD5 内把 sessionRanges 与 DB 已有 ranges 合并，写回 cachedRangesString，更新 lastAccessTime；若已覆盖整首则在主线程调 finishCacheForURL。 |
| `- cachedRangesCoverFullLengthForURL:` | 判断合并后 ranges 是否覆盖 [0, totalLength)。 |
| `- finishCacheForURL` | 在 queueForMD5 内：.tmp 移动为 .mp3，DB 中 isFinished=YES，发 NLCacheManagerDidFinishCachingNotification，异步调 cleanCacheWithMaxSize。 |
| `- cleanCacheWithMaxSize:` | 后台队列：按 lastAccessTime 排序，逐个删文件并删表记录直到总大小≤maxSize（500MB）。 |
| `- clearAllCache` | 删除 NLAudioCache 目录下全部文件，deleteFromTable 清空账本，主线程发 FinishCaching 通知。 |
| `- parseRangesFromString:` / `- stringFromRanges:` | JSON 与 NSArray<NSValue(NSRange)> 互转。 |
| `- mergeRanges:` | 区间排序并合并重叠/相邻。 |

### 2. NLResourceLoader.m（AVPlayer 资源加载 + 缓存）

实现 `AVAssetResourceLoaderDelegate`，将 `http(s)` 替换为 `streaming` 协议，按 Range 请求。

| 方法 | 作用 |
|------|------|
| `- init` | 创建 NSURLSession、taskRequestMap、tasksMap、taskContextMap、mapLock。 |
| `- invalidateSession` | invalidateAndCancel session。 |
| `- resourceLoader:shouldWaitForLoadingOfRequestedResource:` | 根据 requestedOffset/requestedLength 查 NLCacheManager cachedRangesForURL；若命中则从 tempFilePath 或 cacheFilePath 用 pread 读数据 respondWithData，满足则 finishLoading；若不命中或仅部分命中则发起网络请求（Range），数据回调里写 NLCacheManager cacheData 并 respondWithData，完成后 mergeAndSaveSessionRanges、判断 finishCacheForURL。 |

即：播放时先查本地 ranges，命中则直接读文件喂给 AVPlayer；未命中或部分命中则网络请求并写入缓存。

## 五、预加载（下一首）

### NLPlayerManager.m 中与预加载相关

| 属性 | 含义 |
|------|------|
| `nextResourceLoader` | 下一首的 NLResourceLoader。 |
| `nextPlayerItem` | 下一首的 AVPlayerItem（streaming + ResourceLoader）。 |
| `preloadedNextIndex` | 已预加载的下一首索引，-1 表示未预加载。 |

| 方法 | 作用 |
|------|------|
| `- preloadNextItemIfNeeded` | 先 `clearPreloadedItem`；用 `indexForNextInMode:` 算下一首索引 nextIdx；若 nextSong 无 playURL 则 return；将 URL 改为 scheme `streaming`，创建 AVURLAsset + NLResourceLoader（originURL=原 URL），setDelegate:loader，创建 AVPlayerItem，赋给 nextResourceLoader、nextPlayerItem、preloadedNextIndex。 |
| `- clearPreloadedItem` | nextResourceLoader、nextPlayerItem 置 nil，preloadedNextIndex=-1。 |
| `- switchToPreloadedItemAndPlay:` | 若 index==preloadedNextIndex 且 nextPlayerItem 存在：currentIndex/currentSong 更新，addPlayHistory，当前 resourceLoader 换为 nextResourceLoader，player replaceCurrentItemWithPlayerItem:nextPlayerItem，清空预加载引用，移除旧 item 观察、添加新 item 观察与 endTime 观察，play，postSongChangedNotification，再调 preloadNextItemIfNeeded 预加载再下一首。 |
| `- playSongAtIndexIfNeeded:` | 若 index==preloadedNextIndex 且 nextPlayerItem 存在则直接 `switchToPreloadedItemAndPlay:`；否则 clearPreloadedItem，有 playURL 则 playSongAtIndex，无则先 fetchPlayableURL 再播。 |
| `- indexForNextInMode:` | 列表循环/随机/单曲循环下计算下一首索引。 |

流程：每次播放或切歌成功后调 `preloadNextItemIfNeeded` 预加载下一首的 AVPlayerItem（走 streaming+缓存）；切下一首时若命中预加载则 `switchToPreloadedItemAndPlay` 直接替换 item 播放，实现丝滑切歌。

## 八、高度自适应

### 1. 首页 TableView 行高

- `tableView.rowHeight = UITableViewAutomaticDimension`，`estimatedRowHeight = 220`。
- 每个 section 的 cell（NLHomeSectionCell 子类）用 Auto Layout：标题 + collectionView（高度由 config 里按 sizeForCollectionCell 更新）；折叠时 collectionView 高度约束为 0，展开为固定高度，从而整体高度变化。

### 2. 歌单详情页 TableView Header 高度（NLSongListHeaderView）

| 文件 | 方法 | 作用 |
|------|------|------|
| NLSongListViewController.m | `setupTableHeader` | 创建或复用 headerView，configWithPlayList:header，再 `updateHeaderViewHeight`。 |
| NLSongListViewController.m | `updateHeaderViewHeight` | 宽度取 tableView；先设 headerView.bounds 为 (w, largeHeight)，layoutIfNeeded；用 `systemLayoutSizeFittingSize:CGSizeMake(w, UILayoutFittingCompressedSize.height)` 得到 fitSize；若未展开则 headerHeight 限制为屏高 0.75；最后设 headerView.frame 并重新赋值 tableHeaderView，layoutIfNeeded。 |
| NLSongListHeaderView.m | `expandDescTapped` | 切换 _descExpanded，descLabel numberOfLines 展开为 0、收起为 3，按钮文字「更多/收起」，调 delegate headerViewDidRequestRelayout:。 |
| NLSongListHeaderView.m | `updateExpandButtonVisibility` | 用 boundingRectWithSize 测简介是否超过 3 行，决定是否显示「更多」按钮及按钮高度（0 或 28）。 |
| NLSongListHeaderView.m | `configWithPlayList:` | 设置封面、标题、副标题、简介；hideDescription 时简介区约束为 0；否则 set fullDescText、numberOfLines、updateExpandButtonVisibility。 |

即：Header 用 Auto Layout + systemLayoutSizeFittingSize 得到高度；简介「更多/收起」通过 numberOfLines 与 delegate 触发的 updateHeaderViewHeight 实现高度自适应。

| `setComment:` | 赋值 _comment，调 updateContentLabel，设置昵称、时间、点赞、头像。 |
| `updateContentLabel` | 若 comment 尚无 collapsedAttr/expandedAttr：调 NLCommentTextFolder textNeedExpand:width:font 得 needExpand；不需要展开则整段普通富文本；需要则 collapsedAttr = collapsedText、expandedAttr = expandedText。contentLabel 根据 isExpanded 显示对应 attributedText。 |
| `textTap` | 切换 comment.isExpanded，updateContentLabel，执行 expandBlock（让 TableView beginUpdates/endUpdates）。 |

### 3. NLCommentTextFolder.m（富文本折叠/展开）

| 方法 | 作用 |
|------|------|
| `+ textNeedExpand:width:font:` | 用 NSTextStorage + NSLayoutManager + NSTextContainer（宽度 width、高度 CGFLOAT_MAX）布局，统计行数，return lineCount > 3。 |
| `+ collapsedText:font:width:` | 同上 TextKit 布局，取前 3 行对应字符，拼接「...展开」，后段设为蓝色，返回 NSAttributedString。 |
| `+ expandedText:font:` | 全文 + 「 收起」，收起二字蓝色。 |

### 4. NLCommentModel

- isExpanded、needExpand；collapsedAttr、expandedAttr；collapsedHeight、expandedHeight。  
- 高度在 willDisplayCell 时写入，heightForRow 优先用缓存，避免重复计算；展开/收起通过 expandBlock + beginUpdates/endUpdates 只重算该行高度。
