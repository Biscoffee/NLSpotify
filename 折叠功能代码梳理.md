# 首页 TableView 折叠功能 - 代码梳理与讲解

## 一、涉及的文件与职责

| 文件 | 职责 |
|------|------|
| **NLHomeViewController.m** | 维护折叠状态、决定行高、响应“点击标题”并刷新 |
| **NLHomeSectionCell.h / .m** | 提供标题区 UI、点击区域、箭头图标、根据 `collapsed` 显隐列表；点击时通过 block 回调 |
| **NLPlaylistCell** | 继承 NLHomeSectionCell，歌单小卡片（标题 + 横向列表） |
| **NLSingerAlbumCell** | 继承 NLHomeSectionCell，歌手专辑大卡片（头像/用户名 + 横向列表） |

---

## 二、NLHomeViewController（控制器）

### 2.1 状态：谁被折叠了

```objc
@property (nonatomic, strong) NSMutableSet<NSNumber *> *collapsedSections;
```

- **含义**：里面存的是「已折叠的 section 的 index」。
- **例如**：`[@1, @3]` 表示第 1、第 3 个 section 是折叠的，其它是展开的。
- **初始化**：在 `setupUI` 里 `self.collapsedSections = [NSMutableSet set]`，保证不为 nil；`loadData` 里再兜底一次 `if (!self.collapsedSections) self.collapsedSections = [NSMutableSet set]`。

---

### 2.2 配置 Cell 时：把“当前是否折叠”和“点标题回调”交给 Cell

在 `tableView:cellForRowAtIndexPath:` 里，对 **Small** 和 **Big** 两种 cell 都做了同样三件事：

```objc
cell.sectionIndex = indexPath.section;                                    // ① 告诉 cell 自己是第几个 section
cell.collapsed = [self.collapsedSections containsObject:@(indexPath.section)];  // ② 当前是折叠还是展开
cell.didTapHeader = ^(NSInteger sectionIndex) {                            // ③ 点击标题时回调
    __strong typeof(weakSelf) self = weakSelf;
    if (self) [self toggleSectionCollapsed:sectionIndex];
};
```

- **① sectionIndex**：Cell 被点击时会把该值通过 block 参数传给 controller。
- **② collapsed**：会触发 Cell 的 `setCollapsed:`，里面会隐藏/显示横向列表、切换箭头。
- **③ didTapHeader**：点标题时 Cell 调用 `self.didTapHeader(self.sectionIndex)`，controller 在 block 里收到 section 并执行 `toggleSectionCollapsed:`。

---

### 2.3 行高：折叠 52pt，展开按样式 217 / 330

```objc
- (CGFloat)tableView:(UITableView *)tableView heightForRowAtIndexPath:(NSIndexPath *)indexPath {
    if ([self.collapsedSections containsObject:@(indexPath.section)]) {
        return 52;   // 折叠：只显示标题区
    }
    NLSectionViewModel *sectionVM = [self.homeVM sectionAtIndex:indexPath.section];
    switch (sectionVM.style) {
        case NLHomeSectionStylePlaylist:    return 217;
        case NLHomeSectionStyleSingerAlbum: return 330;
        default: return 240;
    }
}
```

- 若该 section 在 `collapsedSections` 里 → 行高 52。
- 否则按 Small/Big 等样式返回 217 / 330 / 240。

---

### 2.4 点击标题后的处理：切换状态并刷新该行

```objc
- (void)toggleSectionCollapsed:(NSInteger)section {
    if (section < 0) return;
    NSNumber *key = @(section);
    if ([self.collapsedSections containsObject:key]) {
        [self.collapsedSections removeObject:key];   // 当前是折叠 → 改为展开
    } else {
        [self.collapsedSections addObject:key];     // 当前是展开 → 改为折叠
    }
    NSIndexPath *indexPath = [NSIndexPath indexPathForRow:0 inSection:section];
    [self.tableView reloadRowsAtIndexPaths:@[indexPath] withRowAnimation:UITableViewRowAnimationFade];
}
```

- 参数为 `NSInteger section`，由 Cell 的 block 回调传入 `sectionIndex`。
- 根据当前是否在 `collapsedSections` 里，对该 section 做「移除/加入」实现切换。
- 只刷新该 section 的那一行，带动画，TableView 会重新问 `heightForRow` 和 `cellForRow`，Cell 会重新拿到新的 `collapsed` 并更新 UI。

---

## 三、NLHomeSectionCell（基类 Cell）

### 3.1 与折叠相关的属性（.h）

```objc
@property (nonatomic, assign) BOOL collapsed;       // 当前是否折叠
@property (nonatomic, assign) NSInteger sectionIndex;  // 所在 section 的 index
@property (nonatomic, copy, nullable) void (^didTapHeader)(NSInteger sectionIndex);  // 点击标题区时回调
```

- **collapsed**：由 controller 在 `cellForRow` 里根据 `collapsedSections` 设置；设置时会走 `setCollapsed:` 更新 UI。
- **sectionIndex**：由 controller 在 `cellForRow` 里赋值，点击时通过 block 参数传给 controller。
- **didTapHeader**：block 回调，参数为当前 section 的 index，由 controller 在 `cellForRow` 里赋值。

---

### 3.2 私有 UI：可点击的标题区 + 箭头（.m）

```objc
@property (nonatomic, strong) UIButton *headerTapButton;      // 覆盖在标题区上的透明按钮
@property (nonatomic, strong) UIImageView *disclosureImageView; // 右侧箭头
```

- **headerTapButton**  
  - 透明，覆盖 `contentView` 顶部，高度 52pt。  
  - 点击会触发 `headerTapped`，再通过 target-action 调 controller 的 `toggleSectionCollapsed:`。  
  - 这样“点标题 = 点按钮”，不会和下面的横向 CollectionView 冲突。

- **disclosureImageView**  
  - 放在标题区右侧，用系统符号：  
    - 折叠时：`chevron.down`（向下箭头）  
    - 展开时：`chevron.up`（向上箭头）  
  - 在 `setCollapsed:` 里根据 `collapsed` 切换图片。

---

### 3.3 点击标题时的逻辑（.m）

```objc
- (void)headerTapped {
    if (self.didTapHeader) self.didTapHeader(self.sectionIndex);
}
```

- 直接调用 block，并传入当前 `sectionIndex`，由 controller 在 block 里执行 `toggleSectionCollapsed:`。

---

### 3.4 根据 collapsed 更新 UI（.m）

```objc
- (void)setCollapsed:(BOOL)collapsed {
    _collapsed = collapsed;
    self.collectionView.hidden = collapsed;   // 折叠时隐藏横向列表
    UIImage *img = collapsed ? [UIImage systemImageNamed:@"chevron.down"] : [UIImage systemImageNamed:@"chevron.up"];
    if (img) self.disclosureImageView.image = [img imageWithRenderingMode:UIImageRenderingModeAlwaysTemplate];
}
```

- **折叠**：隐藏 `collectionView`，箭头改为向下。
- **展开**：显示 `collectionView`，箭头改为向上。
- 每次 controller 在 `cellForRow` 里设置 `cell.collapsed = ...` 时都会走进这里，所以刷新一行后 UI 会跟状态一致。

---

## 四、整体流程串起来

1. **初始 / 刷新**  
   - Controller 在 `cellForRow` 里根据 `collapsedSections` 设置 `cell.sectionIndex`、`cell.collapsed`、`cell.target`、`cell.action`。  
   - `setCollapsed:` 被调用 → 列表显隐 + 箭头方向更新。  
   - `heightForRow` 根据 `collapsedSections` 返回 52 或 217/330。

2. **用户点标题**  
   - 点的是覆盖在标题上的 `headerTapButton` → `headerTapped`。  
   - `headerTapped` 里执行 `self.didTapHeader(self.sectionIndex)`，即调用 controller 赋值的 block，并传入 section 索引。  
   - Block 内执行 `[self toggleSectionCollapsed:sectionIndex]`。

3. **Controller 处理**  
   - `toggleSectionCollapsed:` 里把该 section 从 `collapsedSections` 加入或移除。  
   - 然后 `reloadRowsAtIndexPaths:` 只刷新这一行。

4. **刷新后**  
   - TableView 再次问 `heightForRow`（得到新的 52 或 217/330）和 `cellForRow`。  
   - `cellForRow` 里再次设置 `cell.collapsed = [self.collapsedSections containsObject:@(indexPath.section)]`。  
   - Cell 的 `setCollapsed:` 再次执行，列表显隐和箭头与当前状态一致。

---

## 五、Small / Big 子类与折叠的关系

- **NLPlaylistCell**、**NLSingerAlbumCell** 都继承自 `NLHomeSectionCell`。  
- 折叠相关的 UI（`headerTapButton`、`disclosureImageView`）和逻辑（`headerTapped`、`setCollapsed:`、`didTapHeader`）都在基类里。  
- 子类只负责：  
  - 注册并配置自己的 CollectionView Cell（Small/Big 样式）；  
  - 设置 `didSelectPlaylist` / `didSelectSingerAlbum` 等业务回调。  
- Controller 对两种 cell 的折叠配置完全一致：  
  - `sectionIndex`、`collapsed`、`didTapHeader` 的赋值方式相同。

---

## 六、小结

- **状态**：只在 Controller 的 `collapsedSections` 里，用 section 的 index（NSNumber）表示是否折叠。  
- **行高**：由 `heightForRow` 根据 `collapsedSections` 决定（52 vs 217/330）。  
- **UI**：Cell 的 `setCollapsed:` 负责列表显隐和箭头；标题区可点击由 `headerTapButton` 完成。  
- **通信**：Cell 通过 block `didTapHeader` 把「哪个 section 被点了」（sectionIndex）传给 controller，controller 在 block 里用 weakSelf 调用 `toggleSectionCollapsed:`，改状态并刷新该行。

这样就把「谁折叠了、行多高、点标题怎么反应」都串在一起，逻辑集中、职责清晰。
