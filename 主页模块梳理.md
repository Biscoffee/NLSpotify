# 主页模块梳理

## 一、文件结构

### 1. 控制器
| 文件 | 职责 |
|------|------|
| `NLHomeViewController.h/.m` | 主页控制器，管理 tableView、折叠状态、导航栏、抽屉 |

### 2. ViewModel
| 文件 | 职责 |
|------|------|
| `NLHomeViewModel.h/.m` | 主页数据层，调用多个 Service 拉取数据并组装 sections |
| `NLSectionViewModel.h/.m` | 单个分区的 ViewModel（style、title、items） |

### 3. Cell
| 文件 | 职责 |
|------|------|
| `NLHomeSectionCell.h/.m` | 分区基类 Cell，标题 + 横向 CollectionView，支持折叠 |
| `NLPlaylistCell.h/.m` | 歌单小卡片（封面+标题），用于「为你推荐」等 |
| `NLSingerAlbumCell.h/.m` | 歌手专辑大卡片（头像+用户名+封面列表） |
| `NLJJAlbumCell.h/.m` | 与 BigCell 类似，当前未使用 |
| `NLSingerCell.h/.m` | 歌手 Cell，继承 HomeSectionCell |

### 4. Model
| 文件 | 职责 |
|------|------|
| `NLRecommendAlbumListModel.h/.m` | 推荐歌单/专辑模型（封面、名称、playlistId 等） |
| `NLSingerAlbumListModel.h/.m` | 歌手/专辑列表模型（头像、标题、cardId 等） |

### 5. 抽屉
| 文件 | 职责 |
|------|------|
| `NLDrawerViewController.h/.m` | 侧边抽屉控制器 |
| `NLDrawerView.h/.m` | 抽屉 UI |
| `NLDrawerModels.h/.m` | 抽屉数据模型 |

## 二、网络请求（主页相关）

### 1. NLRecommendAlbumListService
- **路径**: `Networking/网易云API/为你推荐/NLRecommendAlbumListService.h/.m`
- **方法**: `fetchRecommendPlayListWithLimit:success:failure:`
- **返回**: `NSArray<NLRecommendAlbumListModel *>`
- **用途**: 获取「为你推荐」歌单列表

### 2. NLSingerAlbumListService
- **路径**: `Networking/网易云API/JJAlbum/NLSingerAlbumListService.h/.m`
- **方法**: `fetchSingerListWithSuccess:failure:`
- **返回**: `NSArray<NLSingerAlbumListModel *>`
- **用途**: 获取歌手/专辑列表（大卡片分区）

### 3. NLChineseSongListService
- **路径**: `Networking/网易云API/为你推荐/NLChineseSongListService.h/.m`
- **方法**: `fetchRecommendPlayListWithLimit:success:failure:`（与 Recommend 相同签名）
- **返回**: `NSArray<NLRecommendAlbumListModel *>`
- **用途**: 获取华语歌单推荐

## 三、命名规范问题与修复

### 已修复
1. **NLSongListServiece** → **NLSongListService**（拼写错误 Serviece → Service）
2. **NLHomeViewModel sectionAtIndex:** 参数 `(NSInteger *)index` → `(NSInteger)index`
3. **Cell 语义化命名**: `NLPlaylistBigCell` → `NLSingerAlbumCell`（歌手专辑），`NLPlaylistSmallCell` → `NLPlaylistCell`（歌单）
4. **CollectionCell**: `PlaylistBigCollectionCell` → `SingerAlbumCollectionCell`，`PlaylistSmallCollectionCell` → `PlaylistCollectionCell`
5. **分区枚举**: `NLHomeSectionStylePlaylistSmall/Big` → `NLHomeSectionStylePlaylist` / `NLHomeSectionStyleSingerAlbum`
6. **回调**: 歌手专辑使用 `didSelectSingerAlbum`，歌单使用 `didSelectPlaylist`
7. **NLPlaylistService**：注释拼写 `NLPlaylsitServiece` → `NLPlaylistService`

### 建议后续处理
- **NLJJAlbumCell**：当前未被使用，可删除或改名为 `NLFeaturedAlbumCell`（若为精选专辑场景）
- **NLHomeViewModel**：`loadDataWithCompletion:` 内有多次重复请求逻辑，可抽取为配置化（section 类型 + Service + 标题）

### 网络请求梳理
详见 **`网络请求服务梳理.md`**，包含所有 Service 的 API、用途及使用场景。

## 四、数据流

```
NLHomeViewController.viewDidLoad
    → loadData
    → NLHomeViewModel.loadDataWithCompletion:
        → dispatch_group 并行请求:
            - NLRecommendAlbumListService.fetchRecommendPlayListWithLimit:8（为你推荐）
            - NLSingerAlbumListService.fetchSingerListWithSuccess:（林俊杰）
            - NLChineseSongListService.fetchHighQualityPlaylistsWithCategory:limit:（华语、欧美、流行）
        → 组装 NLSectionViewModel 数组
        → completion
    → tableView.reloadData
```
